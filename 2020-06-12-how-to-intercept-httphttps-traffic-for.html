Unlike the normal vehicle traffic, data sent and received over the protocol between the computer and the webserver is referred to as the network traffic.


In this blog post let us see how we can intercept the network traffic for Android. 


I have used Burpsuite tool to intercept the https traffic and  Android Studio to create an android emulator. Assuming you all know about Burpsuite and Android Studio.


Let us dive straight into the steps.


 Step 1: Setting up a proxy in Burpsuite

Set the proxy settings as below:

Goto proxy > options > Add

Set Port as 8081 and Bind to address as All interfaces

Note: you can use any port of your choice as long as you make sure to configure the same in your android device as well.



Step 2: Exporting CA Certificate

HTTPS uses an encrypted communication protocol knows as TLS/SSL to make it a secure way of transporting data over the internet. To capture this traffic, we need to have trusted user certificates in the android device.

We can import self-signed proxy certificate from Burpsuite and add it on the Android device.

Under the proxy > options tab 

Click on Import/export CA certificate button

In the next window, select Certificate in DER format and click on next button.

In the next window, click on Select file button and choose the location you wish to save the certificate and name the file as Burp.cer( you can name it anything but make sure to use .cer extension for certificate format)




I have saved it on my Desktop. 

Step 3: Adding the certificate to Android device

Now  'drag n drop' the certificate on to Android Emulator I have created using Android studio.




Click on the certificate.

In the pop-up box, under certificate name type in a name for the certificate.

Under Credential Use select Wi-FI or VPN and apps according your requirement. I would recommend to do both and click ok.




Now you can see the burp is installed message displayed. 


You can always check the certificate at the below location.

Note: This location may differ based on the Android version you are using.

Goto Settings > Security > Encription & Credentials > Trusted Credentials > User







Step 4: Configure proxy settings on your Android device

To set up proxy on Android

Goto Wi-FI settings > click on edit > Advanced options

Under Proxy >  Select Manual

Set the Proxy Hostname: Ip address of your host PC

mine is: 192.168.1.3

(You can get this info from CMD > ipconfig)

Set the Proxy Port: 8081

Click on Save button




Step 5: Capture the traffic

Now let us intercept the traffic while searching twitter.com on google chrome browser of your android device as an example.

Turn the intercept on in the Burp proxy as shown.





You are done!

That is how easy it is to intercept HTTPS traffic on Android in 5 steps.
